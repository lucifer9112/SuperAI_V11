# Documentation Coauthoring

## Document Types
| Type | Audience | Length |
|------|---------|--------|
| README | New users | 1-2 pages |
| API docs | Developers | Per endpoint |
| Architecture | Team | 5-10 pages |
| Tutorial | Learners | Step-by-step |
| Changelog | Users | Per release |

## README Template
1. Project name + one-line description
2. Key features (bullet list)
3. Quick start (< 5 steps)
4. Installation
5. Usage examples
6. Configuration
7. Contributing
8. License

## Writing Rules
- Lead with the "what" and "why", then "how"
- Use code examples over prose explanations
- Keep sentences short (< 25 words)
- Use active voice
- Test all code examples before publishing
