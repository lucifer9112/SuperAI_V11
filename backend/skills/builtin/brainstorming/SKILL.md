# Brainstorming

## Process
1. **Define** — what problem are we solving? For whom?
2. **Diverge** — generate as many ideas as possible (no filtering)
3. **Cluster** — group similar ideas into themes
4. **Converge** — select top 3 ideas based on impact vs effort
5. **Refine** — turn selected ideas into concrete next steps

## MVP Framework
- **Must have** — core value proposition (launch with these)
- **Should have** — important but not launch-blocking
- **Could have** — nice additions if time permits
- **Won't have** — explicitly out of scope (for now)

## Questions to Ask
- Who is the user? What's their pain?
- What's the simplest version that delivers value?
- What existing solutions exist? How is this different?
- What's the riskiest assumption? How to test it cheaply?
